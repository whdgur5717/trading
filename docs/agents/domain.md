# 도메인 문서

이 저장소에서 작업할 때 agent와 skill이 도메인 문서를 읽는 기준이다.

## 먼저 읽을 문서

- 루트의 `Glossary.md`
- 루트의 `CONTEXT-MAP.md`: 존재할 때만 읽는다. 이 파일은 맥락별 용어 사전을 가리킨다.
- `docs/adr/`: 작업 영역과 관련된 ADR만 읽는다.

파일이 없으면 조용히 넘어간다. 문서가 없다는 이유만으로 새 문서를 먼저 만들지
않는다.

## 용어 사용 기준

이슈 제목, 리팩터링 제안, 가설, 테스트 이름에서 도메인 개념을 말할 때는
`Glossary.md`에 정의된 용어를 쓴다.

필요한 개념이 `Glossary.md`에 없으면 새 용어를 즉시 만들지 않는다. 먼저 기존
용어로 표현할 수 있는지 확인하고, 실제 공백이면 도메인 용어 정리 작업에서
추가한다.

## ADR 충돌 처리

작업 내용이 기존 ADR과 충돌하면 덮어쓰지 않는다. 어떤 ADR과 왜 충돌하는지 먼저
밝히고, 결정을 다시 열어야 하는 이유를 설명한다.
