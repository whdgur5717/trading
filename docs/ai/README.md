# AI 작업 기록

이 디렉터리는 AI 에이전트가 프로젝트를 이어서 진행하기 위한 작업 기억장입니다.
Codex의 개인 Memories와 별개로, 저장소에 남아야 하는 제품/PM/디자인/기술 맥락만
기록합니다.

## 파일 역할

- `START_HERE.md`: 사람이든 AI든 처음 볼 진입점과 문서 지도.
- `project-context.md`: 현재 프로젝트 운영 맥락과 source of truth 지도.
- `documentation-standards.md`: 작업, 기능, 디자인, 결정 문서 형식.
- `active-context.md`: 현재 이어받아야 할 작업 맥락, 최근 판단, 다음 행동.
- `progress.md`: 완료된 것, 진행 중인 것, 막힌 것, 검증 상태.
- `tasks/_index.md`: 작업 단위 목록과 상태.
- `tasks/TASK-*.md`: 큰 작업의 목적, 계획, 변경, 검증, 남은 리스크.
- `features/`: 기능별 목적, 사용자 흐름, 수용 기준, 데이터/API 고려사항.
- `design/`: 실제 화면/컴포넌트 작업에서 생긴 디자인 판단과 적용 맥락.
- `decisions/`: 되돌아보면 이유가 필요할 제품/디자인/기술 결정.
- `research/`: 조사 결과, 비교, 외부 자료 요약.

## 기록 원칙

- 모든 작업을 장황하게 기록하지 않습니다. 다음 세션에서 맥락이 끊길 작업만 남깁니다.
- 비밀값, 토큰, 계정 정보, 개인 데이터는 기록하지 않습니다.
- 구현 결과보다 결정 이유, 검증 결과, 다음 행동을 우선 기록합니다.
- 완료된 내용은 `active-context.md`에서 줄이고 `progress.md`나 작업 파일로 옮깁니다.
