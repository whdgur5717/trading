# Design Context

이 디렉터리는 현재 UI 작업에서 실제로 적용한 디자인 판단을 저장합니다.
디자인 시스템의 안정적인 원칙은 `DESIGN.md`가 원본입니다.

## 무엇을 어디에 저장하나

- `DESIGN.md`: 색상, 타이포그래피, 표면, 컴포넌트, 레이아웃 원칙.
- `docs/ai/design/*.md`: 특정 화면, 플로우, 컴포넌트에서 내린 디자인 결정.
- `docs/ai/features/*.md`: 디자인 판단이 기능 요구사항이나 수용 기준과 연결될 때.
- `docs/ai/decisions/*.md`: 장기적으로 유지될 디자인 시스템 변경.

## 디자인 노트에 남길 것

- 대상 화면이나 컴포넌트.
- 사용자가 하려는 일과 정보 우선순위.
- `DESIGN.md`에서 적용한 원칙.
- 예외적으로 새 패턴을 쓴 이유.
- 접근성, 반응형, 상태 처리 기준.
- 남은 질문이나 검증 필요 사항.

## 남기지 않을 것

- 단순 CSS 수치 변경 로그.
- 이미 `DESIGN.md`에 있는 토큰의 복사본.
- 임시 시안 설명이나 확정되지 않은 취향 표현.
