# Start Here

이 문서는 새로 들어온 사람이나 AI 에이전트가 프로젝트 맥락을 어디서부터 봐야
하는지 알려주는 진입점입니다.

## 먼저 볼 것

1. `PRODUCT.md`: 제품 목적, 사용자, 톤, 제품 원칙.
2. `DESIGN.md`: 디자인 시스템, 색상, 타이포그래피, 레이아웃, 컴포넌트 원칙.
3. `docs/ai/project-context.md`: 현재 프로젝트 운영 맥락과 문서 지도.
4. `docs/ai/active-context.md`: 지금 이어받아야 할 작업 맥락.
5. `docs/ai/progress.md`: 완료, 진행 중, 막힌 것, 검증 상태.
6. `docs/ai/tasks/_index.md`: 작업 목록.

## 작업별 추가로 볼 것

- 제품/기능: `docs/ai/features/`
- UI/디자인: `docs/ai/design/`
- 기술/구조 결정: `docs/ai/decisions/`
- 조사/비교: `docs/ai/research/`
- 문서 형식: `docs/ai/documentation-standards.md`
- 프론트엔드 작업: `front/AGENTS.md`
- 백엔드 작업: `back/AGENTS.md`

## 기록할 곳

- 지금 맥락과 다음 행동: `docs/ai/active-context.md`
- 진행 상태와 검증 결과: `docs/ai/progress.md`
- 작업 단위 기록: `docs/ai/tasks/TASK-*.md`
- 기능 스펙과 수용 기준: `docs/ai/features/*.md`
- 디자인 방향과 화면/컴포넌트 결정: `docs/ai/design/*.md`
- 장기 결정: `docs/ai/decisions/YYYY-MM-DD-*.md`
- 조사 결과: `docs/ai/research/*.md`

## 원칙

- `PRODUCT.md`와 `DESIGN.md`는 안정적인 원칙 문서입니다.
- `docs/ai/`는 현재 작업과 결정의 운영 기록입니다.
- 같은 내용을 여러 곳에 복붙하지 말고, 원본 문서를 링크합니다.
- 다음 사람이 알아야 할 이유, 상태, 검증 결과를 남깁니다.
